# Мобильная адаптивность

## Обзор

Проект полностью адаптирован для мобильных устройств. Все компоненты имеют responsive дизайн с поддержкой различных размеров экранов.

## Структура мобильных стилей

```
frontend/src/assets/style/mobile/
├── _variables.scss      # Переменные и брейкпоинты
├── _mixins.scss         # Миксины для мобильных устройств
├── _app-view.scss       # Стили для AppView
├── _production-view.scss # Стили для ProductionView
├── _main-page.scss      # Стили для главной страницы
└── _index.scss          # Главный файл импортов
```

## Брейкпоинты

- **Мобильные маленькие**: до 425px
- **Мобильные средние**: 425px - 768px
- **Планшеты**: 768px - 1024px
- **Десктопы**: от 1024px

## Основные изменения для мобильных

### 1. AppView (Страница применения)

- **Секции**: Вертикальная компоновка вместо горизонтальной
- **Изображения**: Адаптивные размеры (300-400px на мобильных)
- **Текст**: Уменьшенные размеры шрифтов
- **Навигация**: Вертикальное меню, полная ширина кнопок

### 2. ProductionView (Страница производства)

- **Блоки**: Вертикальная компоновка
- **Изображения**: Адаптивные размеры с сохранением пропорций
- **Карусель**: Уменьшенные стрелки навигации
- **Кнопки**: Полная ширина на мобильных

### 3. Главная страница

- **Фон**: Адаптивное позиционирование и размер
- **Заголовки**: Уменьшенные размеры шрифтов
- **Иконки**: Адаптивные размеры
- **Социальные иконки**: Обертка для маленьких экранов

### 4. Навигация

- **Бургер-меню**: Адаптивный размер кнопки
- **Меню**: Уменьшенные отступы и размеры
- **Кнопки**: Минимальная высота 44px для удобного нажатия

### 5. Модальные окна

- **Ширина**: 90-95% на мобильных
- **Отступы**: Уменьшенные для экономии места
- **Формы**: Полная ширина полей ввода
- **Кнопки**: Полная ширина на маленьких экранах

## Использование миксинов

### Пример использования миксинов:

```scss
.my-component {
  // Десктоп стили
  width: 1200px;
  padding: 40px;
  
  // Мобильные стили через миксин
  @include mobile {
    width: 100%;
    padding: 15px;
  }
  
  // Маленькие мобильные
  @include mobile-small {
    padding: 10px;
  }
}
```

### Доступные миксины:

- `@include mobile` - для всех мобильных (до 768px)
- `@include mobile-small` - для маленьких мобильных (до 425px)
- `@include mobile-medium` - для средних мобильных (425px - 768px)
- `@include tablet` - для планшетов (768px - 1024px)
- `@include desktop` - для десктопов (от 1024px)
- `@include responsive-font($min, $max)` - адаптивный размер шрифта
- `@include responsive-padding($mobile, $tablet, $desktop)` - адаптивные отступы
- `@include mobile-vertical` - вертикальная компоновка на мобильных

## Рекомендации

### 1. Минимальные размеры для нажатия

Все интерактивные элементы (кнопки, ссылки) должны иметь минимальную высоту **44px** для удобного нажатия на мобильных устройствах.

### 2. Адаптивные изображения

Все изображения должны использовать:
- `max-width: 100%`
- `height: auto`
- `object-fit: contain` для сохранения пропорций

### 3. Вертикальная компоновка

На мобильных устройствах предпочтительна вертикальная компоновка элементов:
- Flexbox с `flex-direction: column`
- Полная ширина элементов (`width: 100%`)

### 4. Оптимизация текста

- Используйте адаптивные размеры шрифтов
- Увеличьте межстрочный интервал для читаемости
- Используйте `word-wrap: break-word` для длинных слов

### 5. Производительность

- Используйте `-webkit-overflow-scrolling: touch` для плавной прокрутки
- Минимизируйте использование `transform` и `opacity` для анимаций
- Используйте `will-change` для элементов с анимациями

## Известные проблемы и решения

### 1. Горизонтальный скролл

**Проблема**: Появление горизонтального скролла на мобильных

**Решение**: 
```scss
* {
  max-width: 100%;
  box-sizing: border-box;
}
```

### 2. Фиксированная высота секций

**Проблема**: Секции с `height: 100vh` могут быть слишком высокими на мобильных

**Решение**: Использовать `min-height: 100vh` вместо `height: 100vh`

### 3. Маленькие кнопки

**Проблема**: Кнопки слишком маленькие для нажатия

**Решение**: Минимальная высота 44px и достаточный padding

## Обновление стилей

При добавлении новых компонентов:

1. Используйте миксины из `_mixins.scss`
2. Добавьте мобильные стили в соответствующий файл
3. Протестируйте на разных размерах экранов
4. Убедитесь, что все интерактивные элементы имеют минимальный размер 44px
