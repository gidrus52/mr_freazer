# Админ панель

Простая веб-интерфейс для управления категориями и товарами через API.

## Файлы

- `login.html` - страница входа в систему
- `index.html` - админ панель с модальными окнами

## Как использовать

### 1. Запуск сервера

Убедитесь, что ваш NestJS сервер запущен на порту 3000:

```bash
npm run start:dev
```

### 2. Открытие страницы входа

Откройте файл `login.html` в браузере или запустите локальный сервер:

```bash
# Если у вас установлен Python
python -m http.server 8080

# Или используйте любой другой локальный сервер
```

### 3. Вход в систему

1. Откройте `login.html`
2. Введите email и пароль администратора
3. Нажмите "Войти"
4. Токен будет автоматически сохранен в localStorage

### 4. Использование админ панели

1. Перейдите на страницу `index.html`
2. Нажмите кнопку "Добавить категорию" или "Добавить товар"
3. Заполните форму в модальном окне
4. Нажмите "Создать"

## Функции

### Добавление категории
- Название категории (обязательно)
- Описание (необязательно)
- Родительская категория (необязательно)

### Добавление товара
- Название товара (обязательно)
- Описание (необязательно)
- Цена (обязательно)
- Категория (обязательно)
- Количество на складе (необязательно)
- Изображение товара (необязательно):
  - **Загрузка файла**: Выберите файл изображения (JPEG, PNG, GIF)
  - **URL изображения**: Введите ссылку на изображение

## Требования

- Роль ADMIN для создания категорий и товаров
- Валидный JWT токен
- Сервер должен быть запущен на `http://localhost:3000`

## Устранение неполадок

### Ошибка "Необходима авторизация"
- Убедитесь, что вы вошли в систему через `login.html`
- Проверьте, что токен сохранен в localStorage

### Ошибка сети
- Проверьте, что сервер запущен на порту 3000
- Убедитесь, что CORS настроен правильно

### Ошибка 403 Forbidden
- Убедитесь, что у пользователя есть роль ADMIN
- Проверьте, что токен не истек

## API Endpoints

- `POST /api/auth/login` - вход в систему
- `GET /api/categories` - получение списка категорий
- `POST /api/categories` - создание категории (требует ADMIN)
- `GET /api/products` - получение списка товаров
- `POST /api/products` - создание товара (требует ADMIN)
